# 🧠 Context Saver — Процедура /new

## Когда запускать
При получении `/new` или `/reset` в любом агенте (main, second-brain, etc.)

## Что делать

### Шаг 1: Извлечь ключевую информацию из предыдущей переписки
Анализировать последние ~50 сообщений сессии и искать:

**IP-адреса:**
- Формат: `192.168.x.x`, `10.x.x.x`, внешние IP
- Контекст: что за устройство, для чего

**Токены и ключи:**
- API keys, токены, пароли (показывать частично: первые 3-4 символа + *** + последние 2-3)
- GitHub tokens, Home Assistant tokens, Яндекс токены, etc.

**Устройства:**
- Новые умные устройства (весы, пылесос, камеры, датчики)
- Модель, IP, способ управления

**Решения и договорённости:**
- Что решили, что отложили
- Кто с кем о чём договорился
- Дедлайны и обещания

**Статусы:**
- Проекты, задачи, их прогресс
- Что запущено, что упало, что в процессе

### Шаг 2: Сохранить в vault

Путь: `memory/Контексты/YYYY-MM-DD.md`

Создать файл с датой, структура:
```markdown
---
description: "Контекст сессии от YYYY-MM-DD HH:MM"
type: note
tags: [context, session]
created: YYYY-MM-DD
---

# Контекст сессии — YYYY-MM-DD HH:MM МСК

## Извлечённые данные

### 🔗 IP адреса
| Устройство | IP | Контекст |
|---|---|---|
| ... | ... | ... |

### 🔑 Токены
| Сервис | Фрагмент | Контекст |
|---|---|---|
| ... | ... | ... |

### 📱 Устройства
| Устройство | Модель/IP | Управление |
|---|---|---|
| ... | ... | ... |

### ✅ Решения
- ...

### 📌 Ожидающее
- ...

## Источник
Сессия: [сессия key]
Агент: [agent name]
```

### Шаг 3: Обновить существующие заметки
Если найдено что-то новое:
- Добавить в `40 Объекты/устройства.md`
- Добавить в `MEMORY.md` если долгосрочное
- Добавить в `memory/decisions/` если решение

### Шаг 4: Удалить личное
Если в переписке были персональные данные (телефон, паспорт, etc.) — создать заметку в `00 Входящие/` под заголовком "Удалить после обработки"
