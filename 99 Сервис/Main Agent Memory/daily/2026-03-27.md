## 07:48 МСК - Разблокировка Instagram на роутере OpenWrt

### Проблема
Instagram не открывался — провайдер (46.61.250.142/143) блокировал на уровне DNS, возвращал 127.0.0.1.

### Что сделано
1. **DNS AdGuard** — добавил whitelist `@@||instagram.com^` (но это не помогло полностью)
2. **dnsmasq upstream** — переключил с провайдерских DNS на 8.8.8.8/1.1.1.1 (`noresolv=1`)
3. **dnsmasq → AdGuard** — направил dnsmasq через AdGuard (порт 5353) для фильтрации
4. **V2rayA transparent proxy** — поменял режим с `gfwlist` на `default`, добавил routing rules для transparent трафика
5. **Российский список** — скачал geosite.dat + geoip.dat от runetfreedom (62MB+20MB) в /tmp, перепривязал симлинки v2raya
6. **IPv6 проблема** — клиенты получали публичные IPv6 (2a02:ad8:44fa:4c00::/60), Instagram резолвился в IPv6 Facebook и шёл мимо прокси
7. **Отключил IPv6 раздачу** — `uci set dhcp.lan.ra='disabled'`, `dhcpv6='disabled'`
8. **disable_ipv6 в AdGuard** — блокирует AAAA записи
9. **Отключил кэш dnsmasq** — `cache-size=0`, теперь все запросы видны в AdGuard статистике

### Итог
- Instagram работает для всех устройств в сети
- Трафик: IPv4 → iptables REDIRECT → xray port 52345 → VLESS 🇳🇱 216.57.105.95
- AdGuard показывает все DNS запросы

### Роутер
- OpenWrt 24.10.5
- IP: 192.168.1.1, пароль: root/13spartahN
- AdGuard: порт 3000 (admin/13spartahN), DNS порт 5353
- V2rayA: порт 2017, VLESS Reality xhttp → 216.57.105.95:443
- Российский geosite.dat: /tmp/geosite.dat (runetfreedom, теги: ru-blocked, ru-blocked-all)

### Открыто
- geosite.dat в /tmp — после перезагрузки роутера файл пропадёт! Нужно добавить скрипт автозагрузки или освободить место на /overlay
