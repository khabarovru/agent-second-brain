# 2026-03-18 - Настройка d-brain бота

## Сессия: 05:30 - 07:23 UTC

### Контекст
Денис запустил OpenClaw Gateway впервые, начал настройку d-brain (Telegram-бота для голосового дневника с обработкой через OpenClaw + Todoist).

### Основные достижения

1. **Базовая настройка OpenClaw**
   - Модель по умолчанию: сменил с `claude-sonnet-4-6` → `claude-3-5-haiku-20241022` (экономия ~12x)
   - Fallback цепочка: haiku → sonnet-4-5 → openrouter/free
   - Telegram меню очищено (отключены нативные команды)

2. **Запуск d-brain бота**
   - Восстановлен сломанный `bot/main.py` (добавлен `run_bot()`)
   - Зарегистрированы все handlers: commands, voice, text, photo, forward, do, process, weekly
   - Голосовые расшифровываются через Deepgram → сохраняются в `vault/daily/`

3. **Изоляция обработки**
   - Создан агент `dbrain-processor` (изолированный workspace)
   - Исправлена session lock problem (запросы попадали в webchat)
   - Убран `--local` флаг (теперь через Gateway с изолированной сессией)

4. **MCP Todoist интеграция**
   - Исправлена переменная: `TODOIST_API_KEY` → `TODOIST_API_TOKEN`
   - Скопирован `mcp-config.json` в `~/.openclaw/agents/dbrain-processor/`
   - Исправлен путь к skill: `.openclaw/skills` → `.claude/skills`

5. **Удаление git commit**
   - Убраны `git.commit_and_push()` из `/process` и `/weekly`
   - Проект теперь полностью локальный (без push на GitHub)

### Проблемы и решения

| Проблема | Решение |
|----------|---------|
| Session file locked | Создан изолированный агент `dbrain-processor` |
| MCP недоступен | `TODOIST_API_TOKEN` + правильный путь к skill |
| Git commit failure | Убраны автокоммиты |
| Команды попадали в webchat | `--agent dbrain-processor --session-id dbrain-isolated` |
| `claude-haiku-4` not found | Исправлено на `claude-3-5-haiku-20241022` |
| Модель слишком дорогая | Haiku вместо Sonnet (~12x экономия) |

### Чему научился
1. OpenClaw агенты изолируются через `agents.list` + отдельный workspace
2. MCP конфигурация per-agent через копирование `mcp-config.json`
3. Session lock решается изоляцией агента, а не `--local` флагом
4. `.claude/skills` — стандартная структура для Claude Code/Codex скиллов
