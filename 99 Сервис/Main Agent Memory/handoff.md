# Handoff - 2026-03-26 23:00 MSK

## Тема
Настройка второго агента — секретарь для документооборота (договоры, акты, таблицы).

## Решения
- Создали отдельную папку ~/.openclaw/agents/secretary/ с полной изолированной структурой
- Выбрали модель claude-sonnet-4-6 (не opus) — быстрая и дешёвая для рутины
- Структура агента:
  - inbox/ — входящие файлы (Telegram/ручное)
  - output/ — отчёты агента
  - templates/ — эталонные договоры/акты для сверки
  - scripts/ — parse_pdf.py, parse_docx.py, parse_xlsx.py (Python)
  - skills/ — скиллы секретаря
  - memory/ — lessons.md, patterns.md, projects-log.md

- `~/.openclaw/agents/secretary/agent/inbox/` - точка входа документов

## Контекст
Вопрос был: "Как настроить второго агента для работы с документами?"

Решение: полная изоляция через отдельную папку ~/.openclaw/agents/secretary/, собственный SOUL.md (педантичный секретарь), TOOLS.md с парсерами, AGENTS.md с правилами обработки документов. Агент будет принимать PDF/DOCX/XLSX через inbox/, парсить их скриптами, сверять с шаблонами из templates/, находить ошибки/несоответствия, составлять отчёты в output/.

Модель: claude-sonnet-4-6 вместо opus — потому что рутинная работа (парсинг, сверка, отчёты) не требует глубокого мышления, а скорость и цена важны.

Финальная структура подтверждена и готова к использованию.
