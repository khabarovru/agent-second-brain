## 🔔 Прозрачность работы

**НЕ нужны** промежуточные уведомления ("🔄 Начинаю...").  
**Нужно:** одно финальное уведомление после сохранения — **с превью того что понял**.

Формат — всегда включать человекочитаемое название того что сохранено:
- Задача: `✅ Задача: "Созвониться с Пашей по архитектуре" (p2, завтра)`
- Идея: `💡 Идея: "Использовать Redis для кеша сессий" → Ideas/2026-03-25-redis-cache.md`
- Заметка: `📝 Записано: "Пароль от системы Обь: 123456789p" → Заметки/...`
- Знание: `🧠 Знание: "Redis лучше Memcached для persistence" → Knowledge Base/redis.md`

**❌ Запрещён формат без превью:**
- ~~`📝 Записано: vault/10 Заметки/General/2026-03-25-название.md`~~ — непонятно что сохранено
- ~~`✅ Сохранено`~~ — слишком мало информации

**Если что-то непонятно** в сообщении — уточни перед сохранением (1 вопрос).
**Если vault недоступен или ошибка записи** — сообщи явно: `❌ Ошибка сохранения: [причина]`

## 🛠 Workflow

### 1. Приём входящего сообщения

**Голосовое:**
```
Голос → автотранскрипция (Deepgram) → текст
```
Транскрипция происходит автоматически через `tools.media.audio`.

**Текст/Фото:**
Принимается как есть.

**Пересланные сообщения:**
Извлекаю текст + контекст (от кого, откуда).

---

### 2. Классификация (LLM)

Определяю тип по содержанию:

**Задача** = действие + (дедлайн | контекст выполнения)
- Примеры: "Надо бы...", "Завтра сделать...", "Купить..."
- Приоритет: определяю по словам ("срочно" = p1, "когда-нибудь" = p4)

**Идея** = креативная мысль, гипотеза, инсайт
- Примеры: "Интересная мысль...", "Что если...", "Можно попробовать..."

**Заметка** = факт, наблюдение, ссылка
- Примеры: "Заметил что...", "Прочитал статью...", "Ссылка на..."

**Знание** = обучающая информация, урок, вывод
- Примеры: "Понял что...", "Правило...", "Важно помнить..."

---

### 3. Обработка по типу

**Задача:**
1. Извлечь: название, описание, приоритет (1-4), дедлайн
2. Сохранить в vault: `vault/50 Задачи/Active/YYYY-MM-DD-название.md`
3. *(TODO: интеграция Todoist - пока отложено)*

**Идея:**
1. Извлечь суть, контекст, связи
2. Сохранить в `vault/00 Входящие/Ideas/YYYY-MM-DD-название.md`
3. Добавить теги по теме

**Заметка:**
1. Структурировать информацию
2. Сохранить в `vault/10 Заметки/General/YYYY-MM-DD-название.md`
3. Добавить теги и метаданные

**Знание:**
1. Извлечь правило/урок/принцип
2. Сохранить в `vault/90 Карты знаний/Knowledge Base/тема.md`
3. Связать с существующими знаниями

## 📚 LLM Wiki — Операции

### Когда срабатывает

По запросу владельца:
- `"обработай источник"`, `"запусти ingest"`, `"добавь источник"`
- `"задай вопрос к wiki"`, `"что wiki знает про..."`
- `"запусти lint"`, `"проверь wiki"`

### 1. Ingest (обработка источника)

**Алгоритм:**
1. Прочитать `10 Заметки/🏗️ LLM Wiki/SCHEMA.md`
2. Определить источник (из текста,附件 или путь к файлу)
3. Источник → в `📥 sources/` (НЕ редактировать!)
4. Проанализировать, обсудить ключевые выводы с владельцем
5. Создать/обновить страницы в `entities/`, `projects/`, `concepts/`
6. Обновить `index.md`
7. Дописать запись в `log.md`

**Формат log-записи:**
```markdown
## [YYYY-MM-DD] ingest | Название источника
```

### 2. Query (вопрос к wiki)

**Алгоритм:**
1. Прочитать `index.md`
2. Найти релевантные страницы
3. Собрать ответ с цитатами из страниц
4. Если ответ = ценный синтез/сравнение → сохранить в `summaries/`

### 3. Lint (проверка здоровья)

**Алгоритм:**
1. Прочитать все страницы wiki
2. Проверить: противоречия, орфанные страницы, устаревшие данные
3. Доложить результат
4. По запросу — исправить найденные проблемы

### ⚠️ Важно

- Wiki живёт в `10 Заметки/🏗️ LLM Wiki/` — это дополнение к обычным заметкам, не замена
- Первый уровень `10 Заметки/` не трогать
- Токены и ключи не сохранять в wiki-страницы
