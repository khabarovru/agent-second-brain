2. **Извлекаю структуру:**

## Команды vs Заметки

**Команда** = начинается с `/` или специальный эмодзи (📊)
- `/status` или `📊 Статус` → показать статистику vault
- `/process` → обработать записи
- `/do <запрос>` → выполнить действие
- `/weekly` → недельный дайджест

**Заметка** = всё остальное
- Голосовое сообщение → сохранить
- Текст без `/` → сохранить
- Фото → сохранить

**НЕ путать!** `📊 Статус` ≠ session_status (это команда vault)

## Отчёты прогресса (ОБЯЗАТЕЛЬНО)

Всегда рапортуй прогресс в формате:
**Статус: [принял/обрабатываю N%/готово] | Задача: [кратко] | Детали:**

**Правила:**
- **Принял**: `Статус: принял | Задача: [описание]. Анализирую файлы...`
- **Планирую**: `Статус: 10% | План: 1.[шаг1] 2.[шаг2] 3.[тест]. Подтверди план.`
- **Код**: `Статус: 40% | Пишу [файл/функцию]. Код ниже...`
- **Тест**: `Статус: 70% | Тестирую: [команда]. Результат: OK/FAIL.`
- **Готово**: `Статус: 100% | Результат: [summary]. Commit message готов.`
- НИКОГДА не молчи >30 сек. Без статуса = ошибка.

## 📚 LLM Wiki — Персональная база знаний (дополнение)

В vault есть папка `10 Заметки/🏗️ LLM Wiki/` — это персональная база знаний, построенная по принципу:
**LLM постепенно создаёт и поддерживает структурированную wiki из markdown-файлов**.

### Отличие от обычных заметок

| Обычная заметка | LLM Wiki |
|---|---|
| Фиксирует мысль | Извлекает знание из источника |
| Изолирована | Связана перекрёстными ссылками |
| Не обновляется | Обновляется при каждом новом источнике |

### Когда затрагивается wiki

Wiki-операции **не противоречат** обычному workflow захвата. Они запускаются **по запросу** владельца:

- `"обработай новый источник"` / `"запусти ingest"` → читать `10 Заметки/🏗️ LLM Wiki/SCHEMA.md`
- `"задай вопрос к wiki"` / `"покажи что знаешь про..."` → искать в wiki, отвечать со ссылками
- `"запусти lint"` → проверить здоровье wiki (противоречия, орфанные страницы)

### Структура wiki

```
10 Заметки/🏗️ LLM Wiki/     ← НЕ трогать эту папку
├── SCHEMA.md                  # Правила для агента
├── index.md                  # Каталог страниц
├── log.md                    # Хронология
├── 📥 sources/               # Сырые источники (не редактировать)
├── 🏛️ entities/              # Сущности: объекты, подрядчики
├── 📖 concepts/               # Понятия: ТО, СЗ, диспетчеризация
├── 📊 projects/              # Проекты
└── 📝 summaries/             # Сводки и синтезы
```

### Правила работы с wiki

1. **Перед любой wiki-операцией** — прочитать `SCHEMA.md`
2. **Источники не редактируются** — только читаются
3. **Один источник → много страниц** — после ingest обновить index и log
4. **Перекрёстные ссылки** — использовать `[[wikilink]]` на связанные страницы
5. **Синтез сохранять** — если запрос родил вывод/сравнение/анализ → сохранить в `summaries/`
