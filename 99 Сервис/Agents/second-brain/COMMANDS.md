## /process

**Полный pipeline обработки дня.** Выполняй строго по фазам:

### Фаза 0: PROCESS GOALS (обязательно перед обработкой!)

**Проверь и создай process goals — повторяющиеся ежедневные задачи из целей.**

1. Проверь существующие process goals:
   ```bash
   exec agent/scripts/todoist-find-tasks.sh '{"labels": ["process-goal"]}'
   ```

2. **Если ПУСТО** — прочитай цели и создай сразу (не спрашивай!):
   - `vault/60 Цели/3-weekly.md` → секция `ONE Big Thing` → `2h deep work: {текст}` (every weekday, p2)
   - `vault/60 Цели/2-monthly.md` → топ-приоритет → `1 действие/день: {приоритет}` (every weekday, p3)
   - `vault/60 Цели/1-yearly-2026.md` → фокус года → `30 мин: {фокус}` (every day, p4)

   Создавай ТРЕМЯ отдельными вызовами (или одним пакетным):
   ```bash
   exec agent/scripts/todoist-add-task.sh '{"content": "2h deep work: ...", "due_string": "every weekday at 9am", "priority": 2, "labels": ["process-goal"]}'
   exec agent/scripts/todoist-add-task.sh '{"content": "1 действие/день: ...", "due_string": "every weekday", "priority": 3, "labels": ["process-goal"]}'
   exec agent/scripts/todoist-add-task.sh '{"content": "30 мин: ...", "due_string": "every day", "priority": 4, "labels": ["process-goal"]}'
   ```

   ⚠️ **Правило формулировки:** ПРОЦЕСС, не результат:
   - ❌ "Закрыть акты по Шайбе" → ✅ "Позвонить по актам Шайба/Международная"
   - ❌ "Получить подтверждение" → ✅ "Написать ЦЭПБ Михаилу — статус"
   - ❌ "Запустить Home Assistant" → ✅ "30 мин: настройка HA мониторинга"

3. **Если ЕСТЬ** — проверь статус:
   - Просроченные (due < today) → отметь ⚠️ в отчёте
   - Устаревшие (цели изменились) → рекомендуй удалить
   - Активные → ✅ в отчёт

4. В HTML-отчёте добавь секцию:
   ```html
   <b>📋 Process Goals:</b>
   • 2h deep work: {текст} → ✅ активен
   • 1 действие/день: {текст} → ⚠️ просрочен
   {N} активных | {M} требуют внимания
   ```

### Фаза 1: CAPTURE
1. Запусти скрипт для чтения дневника:
   ```bash
   exec agent/scripts/vault-process.sh
   ```
2. Если скрипт вернул `READY_FOR_PROCESSING: DATE` — читай содержимое дневника
3. Прочитай файл целей:
   - `vault/60 Цели/3-weekly.md` — ONE Big Thing на неделю
   - `vault/60 Цели/2-monthly.md` — месячные приоритеты
4. Следуй инструкциям из `vault/99 Сервис/Skills/dbrain-processor/phases/capture.md`
5. Классифицируй каждую запись: task / idea / reflection / learning / crm_update / skip

### Фаза 2: EXECUTE
1. Следуй инструкциям из `vault/99 Сервис/Skills/dbrain-processor/phases/execute.md`
2. Для каждой записи `classification: "task"` — создай задачу в Todoist:
   ```bash
   exec agent/scripts/todoist-add-task.sh '{"content": "...", "priority": N, "due_string": "..."}'
   ```
3. Для мыслей (idea/reflection/learning) — создай файлы в vault
4. Обнови CRM если нужно

### Фаза 3: REFLECT
1. Следуй инструкциям из `vault/99 Сервис/Skills/dbrain-processor/phases/reflect.md`
2. Сформируй HTML-отчёт по шаблону из `vault/99 Сервис/Skills/dbrain-processor/SKILL.md`
3. Запиши лог в дневник
4. Обнови MEMORY.md если есть значимые изменения

## /weekly

**Цель:** прочитать дневники за неделю, сгенерировать осмысленный дайджест и отправить его владельцу в Telegram.

### Шаг 1: Читай файлы

Определи текущую дату. Прочитай дневники за последние 7 дней:
```
vault/20 Ежедневник/Daily Notes/YYYY-MM-DD.md
```
Читай КАЖДЫЙ файл полностью через `read`. Если файла нет — пропусти.

### Шаг 2: Анализируй

По прочитанному контенту составь структурированный анализ:

- **Главные темы недели** — что было в фокусе? (2-4 темы)
- **Сделано** — конкретные завершённые вещи (список)
- **Идеи и инсайты** — новые мысли, находки, решения
- **Незакрытое** — задачи/вопросы которые остались висеть
- **Одна фраза о неделе** — субъективный итог (как ощущалась неделя)

### Шаг 3: Сохрани в vault

Определи номер недели (ISO week). Запиши полный дайджест в:
```
vault/85 Сводки/YYYY-WNN-summary.md
```

Формат файла — Markdown с заголовками по секциям выше.

### Шаг 4: Отправь владельцу

Сформируй HTML-сообщение и **обязательно отправь через message tool**:

```
message(action="send", target="vk:85950664", channel="vk", message="<текст дайджеста>")
```

Формат сообщения:

```
📅 <b>Недельный дайджест YYYY-WNN</b>
<i>Период: DD.MM — DD.MM.YYYY</i>

🎯 <b>Главные темы:</b>
• тема 1
• тема 2

✅ <b>Сделано:</b>
• конкретная вещь 1
• конкретная вещь 2

💡 <b>Идеи и инсайты:</b>
• инсайт 1

⚠️ <b>Незакрытое:</b>
• задача/вопрос

📝 <b>Итог недели:</b>
одна фраза субъективно

📊 Дней с записями: N
```

⚠️ **Критично:** 
- Вызов `message` — ОБЯЗАТЕЛЬНЫЙ последний шаг, без него команда не выполнена
- Отправляй КОНТЕНТ дайджеста, не метаданные о файле
- Если дней с записями мало — всё равно пиши по тому что есть
- После отправки ответь: `✅ Дайджест W-NN отправлен`

## /email или 📧 Почта

**Что делать:**
1. Запусти проверку почты:
   ```bash
   exec agent/scripts/email-check.sh
   ```
2. Скрипт вернёт JSON. Разбери по полю `emails`:
   - `urgent` → создай задачу в Todoist (p1, deadline: today) для каждого письма
   - `important` → создай заметку в vault/00 Входящие/Email/
   - `info` → сгруппируй в дайджест одной строкой
   - `spam` → если >3 письма от одного отправителя — предложи отписку
3. Если `total: 0` — ответь ТОЛЬКО: `NO_REPLY` (без текста, без уведомлений)
4. Отправь краткий отчёт владельцу
