# SCHEMA — LLM Wiki по лифтам

## Принцип

Wiki строится **между** тобой и сырыми источниками. LLM читает источник → создаёт/обновляет страницы wiki → wiki становится основным инструментом для ответов. Источники не редактируются.

## Структура папок

```
vault/
└── 10 Заметки/           ← НЕ ТРОГАТЬ (существующая структура)
    └── 🏗️ LLM Wiki/      ← Наша wiki (всё внутри)
        ├── 📥 sources/    # Сырые источники (НЕ редактировать)
        ├── 🏛️ entities/  # Сущности: объекты, подрядчики, люди
        ├── 📖 concepts/  # Понятия: ТО, СЗ, диспетчеризация...
        ├── 📊 projects/  # Проекты: замена связи, освидетельствование...
        ├── 📝 summaries/ # Сводки, сравнения, синтезы
        ├── index.md     # Каталог всех страниц wiki
        └── log.md       # Хронология операций
```

> ⚠️ Директория `10 Заметки/` — корневая структура vault. Не переименовывать, не двигать без запроса.

## Типы страниц

### entity/object (Сущность — объект)
```yaml
---
type: entity
subtype: object
created: 2026-04-08
tags: [лифты, объект]
sources: ["SИРИУС АРЕНА — wiki-источник"]
---

# Сириус Арена

## Паспорт
- **Адрес:** ...
- **Лифтов:** 13
- **Подрядчик ТО:** ...
- **Ежемесячная оплата:** ...
```

## Правила

| Правило | Описание |
|---|---|
| **Frontmatter** | Обязательно: `type`, `created`, `tags`, `sources` |
| **Wikilinks** | Все связи — через `[[wikilink]]` |
| **Конфликты** | Если новый источник противоречит старому → отметить явно |
| **One source = many pages** | Один источник может обновить 5–15 страниц |
| **Синтез → summaries/** | Выводы, сравнения, анализ — сохранять в summaries/ |

## Workflow

### Ingest
1. Прочитать источник из `📥 sources/`
2. Обсудить ключевые выводы
3. Создать/обновить страницы
4. Обновить `index.md`
5. Дописать запись в `log.md`

### Query
1. Прочитать `index.md`
2. Найти нужные страницы
3. Ответ с цитатами
4. Если ответ = ценный синтез → сохранить в `summaries/`

### Lint (периодически)
- Противоречия между страницами
- Орфанные страницы (без входящих ссылок)
- Устаревшие данные
- Пробелы → можно заполнить через поиск
